\section{Schlussbetrachtung}

Die vorliegende Untersuchung zeigt theoretische Möglichkeiten auf, eine ressourceneffiziente Videoverarbeitungskette unter Linux zu realisieren. Ein direkter Zugriff auf USB-Kameras ist mittels \enquote{ffmpeg} möglich. Um CPU und RAM zu entlasten, bietet sich die hardwarebeschleunigte Enkodierung mittels \enquote{VA-API} an, welche speziell auf Hardware wie der 
\enquote{AMD Ryzen R1000 Serie} effizient operiert.\newline

Für die Detektion von Spielern stellen \enquote{Histogram of Oriented Gradients} in Verbindung mit \enquote{Support Vector Machines} einen bewährten, rechenarmen Ansatz dar. 
Die anschließende Übertragung an einen Server kann mittels Übertragung über \enquote{REST}-Schnittstellen an einen \enquote{Object-Storage} Server realisiert werden. Andernfalls,
könnten die Videos zur Minimierung von Latenz und Bandbreite über das \enquote{Secure Reliable Transport} Protokoll übertragen werden, welches für instabile Netzwerke optimiert ist.
