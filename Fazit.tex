\section{Schlussbetrachtung}

Die vorliegende Untersuchung zeigt theoretische Möglichkeiten auf, eine ressourceneffiziente Videoverarbeitungskette unter Ubuntu zu realisieren. Ein direkter Zugriff auf USB-Kameras ist über die \enquote{V4L2-Schnittstelle} 
oder mittels \enquote{ffmpeg} möglich. Um CPU und RAM zu entlasten, bietet sich die hardwarebeschleunigte Enkodierung mittels \enquote{VA-API} an, welche speziell auf Hardware wie der 
\enquote{AMD Ryzen R1000 Serie} effizient operiert.\newline

Für die Detektion von Spielern stellen \enquote{Histogram of Oriented Gradients} (HOG) in Verbindung mit \enquote{Support Vector Machines} (SVM) einen bewährten, rechenarmen Ansatz dar. 
Die anschließende Übertragung an einen Server kann zur Minimierung von Latenz und Bandbreite über das \enquote{Secure Reliable Transport} (SRT) Protokoll erfolgen, welches für instabile Netzwerke optimiert ist. Oder
mittels REST-Schnittstellen können die Videos an einen \enquote{Object-Storage} Server übertragen werden.
